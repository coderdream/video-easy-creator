# 解决方案记录（2026011901）

## 背景与目标
- 使用 NVIDIA integrate API 的 minimaxai/minimax-m2.1 模型提供翻译服务
- 参考 TranslateUtil.translateEngSrc 的提示词与分块翻译策略，提示词保持一致

## 实施过程
1. 定位 TranslateUtil.translateEngSrc 的提示词与分块逻辑（编号、行数校验、重试）。
2. 新增 NVIDIA 翻译服务类，使用 Hutool 发起请求并解析返回：
   - 调用 `https://integrate.api.nvidia.com/v1/chat/completions`
   - 模型固定为 `minimaxai/minimax-m2.1`
   - 复用同样的提示词与编号输出约束
3. 补充 `NVIDIA_API_KEY` 常量，便于统一配置。
4. 新增 `requestCompletion` 方法用于连通性测试，并在响应中剔除 `<think>` 思考内容。
5. 调整 NVIDIA 调用加入超时上调与简单重试，降低 429/超时导致的失败。
6. 增加词汇翻译测试用例，校验 36 行输出与首两行原文保持一致；连通性测试保留为手动运行。
7. 新增 Codex API 工具类与常量，支持 OpenAI 兼容请求与多端点尝试。
8. 将 `TranslationUtil.processVoc` 默认提供方切换为 Codex，并保留 Gemini/NVIDIA 可选分支。
9. 新增 Codex 测试用例，API 不可用时自动跳过以避免误报。
10. 增加音标与描述类提示词验证测试，用于评估 NVIDIA 替换可行性。
