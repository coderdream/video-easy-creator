# MiniMax M2.1 API 集成解决方案

## 1. 概述

本文档记录了将 NVIDIA NIM 平台上的 MiniMax M2.1 模型集成到 video-easy-creator 项目的开发过程。MiniMax M2.1 可作为 Gemini API 的替代方案，用于翻译词汇和脚本。

## 2. 技术方案

### 2.1 API 配置

| 配置项 | 值 |
|--------|-----|
| API 地址 | `https://integrate.api.nvidia.com/v1/chat/completions` |
| Model ID | `minimaxai/minimax-m2.1` |
| 认证方式 | Bearer Token |
| 请求格式 | OpenAI 兼容模式 |

### 2.2 核心特性

1. **超大上下文窗口**: MiniMax M2.1 支持 200k+ tokens，适合整篇字幕翻译
2. **思考模式**: 响应中可能包含 `<think>...</think>` 标签，需要过滤
3. **免费额度**: NVIDIA NIM 免费通道有速率限制（约 60 Requests/Minute）

### 2.3 请求格式

```json
{
  "model": "minimaxai/minimax-m2.1",
  "messages": [
    {
      "role": "system",
      "content": "系统提示（可选）"
    },
    {
      "role": "user",
      "content": "用户输入"
    }
  ],
  "temperature": 0.3,
  "top_p": 0.7,
  "max_tokens": 4096,
  "stream": false
}
```

### 2.4 响应格式

```json
{
  "id": "xxx",
  "object": "chat.completion",
  "created": 1768999751,
  "model": "minimaxai/minimax-m2.1",
  "choices": [
    {
      "index": 0,
      "message": {
        "role": "assistant",
        "content": "响应内容"
      },
      "finish_reason": "stop"
    }
  ],
  "usage": {
    "prompt_tokens": 51,
    "total_tokens": 465,
    "completion_tokens": 414
  }
}
```

## 3. 实现细节

### 3.1 文件结构

```
src/main/java/com/coderdream/util/nvidia/
└── MiniMaxUtil.java          # MiniMax API 工具类

src/test/java/com/coderdream/util/nvidia/
└── MiniMaxUtilTest.java      # 测试类
```

### 3.2 核心方法

| 方法名 | 功能 | 参数 |
|--------|------|------|
| `generateContent(String prompt)` | 基本内容生成 | 提示文本 |
| `generateContent(String prompt, String modelName)` | 指定模型生成 | 提示文本, 模型名 |
| `generateContent(String prompt, String modelName, String systemPrompt)` | 带系统提示生成 | 提示文本, 模型名, 系统提示 |
| `translateToChinese(String text)` | 通用翻译 | 英文文本 |
| `translateToChineseForSubtitle(String text)` | 字幕翻译 | 字幕文本 |
| `translateVocabulary(String vocText)` | 词汇翻译 | 词汇文本 |
| `translateScript(String scriptText)` | 脚本翻译 | 脚本文本 |
| `testConnection()` | 测试连接 | 无 |

### 3.3 错误处理

1. **重试机制**: 最多重试 3 次，每次间隔 2 秒
2. **503 过载**: 自动重试
3. **429 速率限制**: 等待更长时间后重试
4. **超时设置**: 120 秒（因为 MiniMax 可能需要较长时间处理）

### 3.4 Think 标签处理

MiniMax M2.1 是支持"思考"的模型，响应中可能包含 `<think>...</think>` 标签。工具类会自动移除这些标签：

```java
private static final Pattern THINK_PATTERN = Pattern.compile("<think>.*?</think>", Pattern.DOTALL);

private static String removeThinkTags(String text) {
    Matcher matcher = THINK_PATTERN.matcher(text);
    String result = matcher.replaceAll("");
    result = result.replaceAll("\\n{3,}", "\n\n").trim();
    return result;
}
```

## 4. 使用示例

### 4.1 基本翻译

```java
String result = MiniMaxUtil.translateToChinese("Hello, world!");
System.out.println(result); // 你好，世界！
```

### 4.2 字幕翻译

```java
String subtitle = "Hello and welcome to 6 Minute English.";
String translated = MiniMaxUtil.translateToChineseForSubtitle(subtitle);
```

### 4.3 词汇翻译

```java
String vocText = "cephalopod\nthe group of animals to which the octopus belongs\n...";
String result = MiniMaxUtil.translateVocabulary(vocText);
```

### 4.4 脚本翻译

```java
String script = "Hello and welcome to 6 Minute English.\nI'm Neil...";
String result = MiniMaxUtil.translateScript(script);
```

## 5. 与 Gemini API 的对比

| 特性 | Gemini API | MiniMax M2.1 |
|------|------------|--------------|
| 上下文窗口 | 1M tokens | 200k+ tokens |
| 中文能力 | 良好 | 优秀（国产模型） |
| 免费额度 | 有限制 | 有限制（60 RPM） |
| 思考模式 | 无 | 有（需过滤标签） |
| 响应速度 | 较快 | 中等 |

## 6. 集成到现有流程

要将 MiniMax 集成到 `SixMinutesStepByStepTest` 中，需要修改以下文件：

### 6.1 修改 TranslateUtil

在 `TranslateUtil.java` 中添加 MiniMax 支持：

```java
// 在 genScriptDialogCn 方法中
if (CdConstants.AI_PROVIDER_NVIDIA.equals(provider)) {
    result = MiniMaxUtil.translateScript(scriptText);
} else {
    result = GeminiApiClient.generateContent(prompt);
}
```

### 6.2 修改 DictUtil

在 `DictUtil.java` 中添加 MiniMax 支持：

```java
// 在 genVocCnWithGemini 方法中
if (CdConstants.AI_PROVIDER_NVIDIA.equals(provider)) {
    result = MiniMaxUtil.translateVocabulary(vocText);
} else {
    result = GeminiApiClient.generateContent(prompt);
}
```

## 7. 注意事项

1. **单线程调用**: 不要多线程高并发调用，避免触发 429 错误
2. **代理配置**: 需要配置代理才能访问 NVIDIA NIM 平台
3. **API Key 安全**: 不要将 API Key 提交到版本控制系统
4. **响应时间**: MiniMax 的响应时间可能较长，已设置 120 秒超时
