## 1. 需求概述

### 1.1 项目背景

为了增强 video-easy-creator 项目的 AI 能力，需要集成 Anthropic 的 Claude API，开发一个通用的 `ClaudeCodeUtil` 工具类，支持与 Claude 模型进行交互，特别是支持工具调用（Tool Use）和扩展思考（Extended Thinking）等高级功能。

### 1.2 核心目标

- 提供统一的 Anthropic Messages API 调用接口
- 支持多个 Claude 模型版本（Sonnet 4.5、Opus 4.5、Haiku 4.5）
- 实现工具调用（Tool Use）机制，支持文件操作、命令执行等
- 支持扩展思考模式（Extended Thinking）以提升推理质量
- 支持上下文管理和多轮对话
- 提供翻译、代码生成等高级应用场景

### 1.3 使用场景

1. **代码生成与修复** - 利用 Claude 的编码能力进行代码生成、重构和 Bug 修复
2. **文本翻译** - 基于资深翻译官 System Prompt 的高质量翻译
3. **文档分析** - 读取和分析项目文档
4. **自动化任务** - 通过工具调用实现自动化的文件操作和命令执行

## 2. 功能需求

### 2.1 基础功能

#### 2.1.1 API 连接管理
- **功能**: 管理 Anthropic API 连接参数
- **需求**:
  - 支持从环境变量读取 `ANTHROPIC_BASE_URL` 和 `ANTHROPIC_AUTH_TOKEN`
  - 支持自定义 API 端点和认证令牌
  - 支持代理配置（国内开发环境）
  - 实现连接超时和重试机制

#### 2.1.2 模型管理
- **功能**: 支持多个 Claude 模型版本
- **需求**:
  - 支持 Claude Opus 4.5 (`claude-opus-4-5-20251101`)
  - 支持 Claude Sonnet 4.5 (`claude-sonnet-4-5-20250929`)
  - 支持 Claude Haiku 4.5 (`claude-haiku-4-5-20251001`)
  - 支持模型版本动态切换
  - 将模型配置存储在 `CdConstants.java` 中

#### 2.1.3 基础消息调用
- **功能**: 发送简单的文本消息并获取响应
- **需求**:
  - 支持单轮对话
  - 支持自定义 System Prompt
  - 支持设置 `max_tokens` 参数
  - 支持设置 `temperature` 参数
  - 返回结构化的响应对象

### 2.2 高级功能

#### 2.2.1 工具调用（Tool Use）
- **功能**: 支持 Claude 调用本地工具
- **需求**:
  - 定义工具集（Tool Definition）
  - 实现工具执行循环（Tool Call Loop）
  - 支持以下工具:
    - `list_files`: 列出目录文件（基于 Hutool FileUtil）
    - `read_file`: 读取文件内容（UTF-8 编码）
    - `write_file`: 写入文件内容（UTF-8 编码）
    - `execute_command`: 执行系统命令（需要白名单限制）
    - `grep`: 文件内容搜索（正则表达式）
  - 支持多轮工具调用
  - 实现工具结果反馈机制

#### 2.2.2 扩展思考模式（Extended Thinking）
- **功能**: 支持 Claude 4.5 的思维链推理
- **需求**:
  - 支持启用/禁用思考模式
  - 支持设置思考预算（`budget_tokens`）
  - 支持提取思考过程和最终答案
  - 支持 `effort` 参数（high/medium/low）

#### 2.2.3 上下文管理
- **功能**: 维护多轮对话的上下文
- **需求**:
  - 实现对话历史存储
  - 支持上下文压缩（处理长对话）
  - 支持会话隔离
  - 支持上下文导出和导入

### 2.3 应用场景

#### 2.3.1 翻译应用
- **功能**: 基于资深翻译官 System Prompt 的高质量翻译
- **需求**:
  - 实现 `translate(String englishText)` 方法
  - 支持技术文档、文学作品、商业文案等多种场景
  - 支持动态 System Prompt 调整
  - 支持翻译质量验证

#### 2.3.2 代码生成应用
- **功能**: 利用 Claude 进行代码生成和修复
- **需求**:
  - 实现 `generateCode(String requirement)` 方法
  - 支持代码审查和优化
  - 支持 Bug 修复建议
  - 支持代码重构

### 3.3 API 规范

#### 3.3.1 请求格式

```
POST {ANTHROPIC_BASE_URL}/v1/messages
Headers:
  - x-api-key: {ANTHROPIC_AUTH_TOKEN}
  - anthropic-version: 2023-06-01
  - content-type: application/json
  - anthropic-beta: structured-outputs-2025-11-13 (可选)

Body:
{
  "model": "claude-opus-4-5-20251101",
  "max_tokens": 4096,
  "temperature": 0.3,
  "effort": "high",
  "thinking": {
    "type": "enabled",
    "budget_tokens": 2048
  },
  "system": "...",
  "messages": [...],
  "tools": [...]
}
```

#### 3.3.2 响应格式

```json
{
  "id": "msg_...",
  "type": "message",
  "role": "assistant",
  "content": [
    {
      "type": "text",
      "text": "..."
    },
    {
      "type": "tool_use",
      "id": "tool_...",
      "name": "read_file",
      "input": {...}
    }
  ],
  "model": "claude-opus-4-5-20251101",
  "stop_reason": "end_turn",
  "usage": {
    "input_tokens": 100,
    "output_tokens": 50
  }
}
```

## 4. 非功能需求

### 4.1 性能需求

- API 调用超时时间: 120 秒（支持深度推理）
- 工具调用循环最大迭代次数: 10 次
- 支持并发调用（线程安全）
- 响应时间: 一般场景 < 10 秒，复杂推理 < 60 秒

### 4.2 可靠性需求

- 实现自动重试机制（最多 3 次）
- 实现错误恢复和降级策略
- 详细的日志记录
- 异常处理和错误提示

### 4.3 安全需求

- API 密钥不能硬编码，必须从环境变量读取
- 工具调用需要白名单限制（特别是 `execute_command`）
- 文件操作需要路径验证
- 支持代理配置用于国内网络环境

### 4.4 可维护性需求

- 代码注释完整，特别是复杂逻辑
- 提供详细的使用文档和示例
- 支持配置化和扩展性
- 遵循项目代码风格（Lombok、链式调用等）

```
你是一位拥有20年经验的资深中英文翻译官，擅长将复杂的英文技术文档、文学作品或商业报告翻译成地道、优美且准确的中文。

- 避免过度使用被动语态
```
