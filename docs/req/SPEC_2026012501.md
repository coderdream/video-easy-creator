```java
import cn.hutool.http.Header;
import cn.hutool.http.HttpRequest;
import cn.hutool.json.JSONArray;
import cn.hutool.json.JSONObject;
import cn.hutool.json.JSONUtil;

public class ClaudeCodeUtil {

    private static final String API_URL = "https://api.anthropic.com/v1/messages";
    private static final String API_KEY = "YOUR_ANTHROPIC_API_KEY"; // 替换为你的Key
    private static final String VERSION = "2023-06-01";
    private static final String MODEL = "claude-3-7-sonnet-20250219";

    /**
     * 发送简单对话
     * @param userContent 用户的问题或指令
     * @return Claude 的回答
     */
    public static String sendMessage(String userContent) {
        // 1. 构建请求体
        JSONObject body = new JSONObject();
        body.set("model", MODEL);
        body.set("max_tokens", 4096);
        
        // 设置系统提示词（模拟 Claude Code 的行为）
        body.set("system", "You are Claude Code, an expert software engineer. " +
                           "Provide concise, functional code snippets and technical explanations.");

        // 构建消息队列
        JSONArray messages = new JSONArray();
        JSONObject message = new JSONObject();
        message.set("role", "user");
        message.set("content", userContent);
        messages.add(message);
        body.set("messages", messages);

        // 2. 发送请求
        String result = HttpRequest.post(API_URL)
                .header(Header.CONTENT_TYPE, "application/json")
                .header("x-api-key", API_KEY)
                .header("anthropic-version", VERSION)
                .body(JSONUtil.toJsonStr(body))
                .execute()
                .body();

        if (responseJson.containsKey("error")) {
            return "Error: " + responseJson.getByPath("error.message");
        }
        
        return responseJson.getByPath("content[0].text").toString();
    }

    public static void main(String[] args) {
```

### 4. 进阶：如何更像“Claude Code”？

如果你想让你的 Util 真正具备 Claude Code 的能力，你需要实现以下进阶功能：

#### A. Tool Use (函数调用)
Claude Code 的本质是通过 **Tool Use** 来读写文件和执行终端命令。
*   在 API 请求中加入 `tools` 数组。
*   定义如 `readFile`, `writeFile`, `listDirectory` 等工具。
*   Hutool 处理响应后，如果发现 `stop_reason` 是 `tool_use`，则在本地执行相应的 Java 代码，并将结果再次发给 Claude。

#### B. Thinking Mode (思维链)
Claude 3.7 支持开启思考模式，这对于复杂代码重构非常有用：
```java
// 在 body 中加入
JSONObject thinking = new JSONObject();
thinking.set("type", "enabled");
thinking.set("budget_tokens", 1024); // 设置思考预算
body.set("thinking", thinking);
```

#### C. 保持上下文 (Session)
Hutool 的 `ClaudeCodeUtil` 应该维护一个 `List<JSONObject>` 来存储历史对话，每次请求时将历史记录全部发送，以实现连续对话。

### 5. 注意事项
1.  **流式响应 (Streaming)**: 如果你希望像 IDE 一样逐字输出，需要使用 Hutool 的 `executeAsync()` 并结合 `HttpDownloader` 或监听输入流，处理 `text/event-stream` 格式。
2.  **超时设置**: 编写代码的推理往往很慢，建议设置较长的超时时间：`.timeout(60000)` (60秒)。
3.  **代理**: 如果你在国内开发，记得设置代理：
    ```java
    HttpRequest.post(API_URL).setHttpProxy("127.0.0.1", 7890)...
    ```

你想深入了解 **Tool Use (工具调用)** 在 Java 中的具体封装实现吗？


要实现真正的 `ClaudeCodeUtil`（具备自动化执行任务能力），**Tool Use (工具调用)** 是最核心的技术。它让 Claude 不再只是“聊天”，而是可以指挥你的 Java 程序去读写文件、查询数据库或执行系统命令。

以下是基于 **Hutool** 实现 Tool Use 的全流程架构和代码实现。

---

### 1. Tool Use 的核心交互流程

1.  **定义工具**：你告诉 Claude 你有哪些函数（比如 `ls`, `readFile`）。
2.  **Claude 决定**：Claude 发现无法直接回答，会返回一个 `tool_use` 类型的 JSON，请求调用某个工具。
3.  **本地执行**：你的 Java 程序根据 Claude 的请求，使用 Hutool 的 `FileUtil` 或 `RuntimeUtil` 执行本地操作。
4.  **返回结果**：将执行结果再次发给 Claude。
5.  **最终回答**：Claude 根据结果给出最终答案。

---

### 2. 定义工具集 (Tool Definition)

我们需要先用 JSON 描述工具。使用 Hutool 的 `JSONUtil` 构建：

```java
public static JSONArray buildTools() {
    JSONArray tools = new JSONArray();

    // 1. 定义读取文件工具
    JSONObject readFile = new JSONObject();
    readFile.set("name", "read_file");
    readFile.set("description", "读取指定路径文件的内容");
    
    JSONObject inputSchema = new JSONObject();
    inputSchema.set("type", "object");
    
    JSONObject properties = new JSONObject();
    properties.set("path", new JSONObject().set("type", "string").set("description", "文件路径"));
    
    inputSchema.set("properties", properties);
    inputSchema.set("required", new String[]{"path"});
    
    readFile.set("input_schema", inputSchema);
    
    tools.add(readFile);
    // 可以继续 add 更多工具，如 write_file, execute_command
    return tools;
}
```

### 3. 实现本地执行逻辑 (Local Execution)

当 Claude 发出请求时，我们需要通过 Java 反射或简单的 `switch-case` 来调用 Hutool：

```java
import cn.hutool.core.io.FileUtil;
import cn.hutool.core.util.RuntimeUtil;

public static String executeTool(String toolName, JSONObject input) {
    switch (toolName) {
        case "read_file":
            String path = input.getStr("path");
            // 使用 Hutool 读取文件
            return FileUtil.readUtf8String(path);
            
        case "execute_command":
            String cmd = input.getStr("command");
            // 使用 Hutool 执行系统命令
            return RuntimeUtil.execForStr(cmd);
            
        default:
            return "Error: Unknown tool";
    }
}
```

### 4. 完整的 `ClaudeCodeUtil` 循环逻辑

Tool Use 通常需要 **多次往返**。下面是一个支持工具调用的完整示例：

```java
import cn.hutool.http.HttpRequest;
import cn.hutool.json.JSONArray;
import cn.hutool.json.JSONObject;
import cn.hutool.json.JSONUtil;

import java.util.ArrayList;
import java.util.List;

public class ClaudeCodeUtil {

    private static final String API_URL = "https://api.anthropic.com/v1/messages";
    private static final String API_KEY = "你的_API_KEY";

    public static void runTask(String userPrompt) {
        // 维护对话上下文
        List<JSONObject> messages = new ArrayList<>();
        messages.add(new JSONObject().set("role", "user").set("content", userPrompt));

        boolean keepRunning = true;
        while (keepRunning) {
            // 1. 构建请求
            JSONObject body = new JSONObject();
            body.set("model", "claude-3-7-sonnet-20250219");
            body.set("max_tokens", 4096);
            body.set("messages", messages);
            body.set("tools", buildTools()); // 传入工具定义

            // 2. 发送请求
            String responseStr = HttpRequest.post(API_URL)
                    .header("x-api-key", API_KEY)
                    .header("anthropic-version", "2023-06-01")
                    .body(JSONUtil.toJsonStr(body))
                    .execute().body();

            JSONObject response = JSONUtil.parseObj(responseStr);
            JSONArray contentArray = response.getJSONArray("content");
            
            // 将 Claude 的回复加入上下文 (必须要这一步，否则后续对话会报错)
            messages.add(new JSONObject().set("role", "assistant").set("content", contentArray));

            // 3. 检查是否有工具调用请求
            boolean toolUsed = false;
            for (int i = 0; i < contentArray.size(); i++) {
                JSONObject block = contentArray.getJSONObject(i);
                if ("tool_use".equals(block.getStr("type"))) {
                    toolUsed = true;
                    String toolName = block.getStr("name");
                    String toolId = block.getStr("id");
                    JSONObject input = block.getJSONObject("input");

                    System.out.println("Claude 请求执行工具: " + toolName + " 参数: " + input);

                    // 4. 执行本地工具
                    String result = executeTool(toolName, input);

                    // 5. 构建工具返回消息 (role 必须为 user)
                    JSONObject toolResult = new JSONObject();
                    toolResult.set("type", "tool_result");
                    toolResult.set("tool_use_id", toolId);
                    toolResult.set("content", result);

                    JSONArray toolResultContent = new JSONArray();
                    toolResultContent.add(toolResult);
                    
                    messages.add(new JSONObject().set("role", "user").set("content", toolResultContent));
                }
            }

            // 如果没有工具调用，说明 Claude 已经给出了最终文本回答
            if (!toolUsed) {
                System.out.println("Claude 最终回答: " + response.getByPath("content[0].text"));
                keepRunning = false;
            }
        }
    }
}
```

### 5. 关键点深度解析

#### 1. 上下文结构 (The List)
Claude 的 API 要求你发送 **所有的** 对话历史。
*   如果 Claude 想要用工具，它会发回一个 `tool_use`。
*   你**必须**把这个 `tool_use` 放在消息历史中。
*   然后你发送一个 `role: "user"` 的消息，内容包含 `type: "tool_result"`。

#### 2. Hutool 的加持
*   **`FileUtil`**: 极大简化了 `read_file`, `write_file`, `list_files` 的实现。
*   **`RuntimeUtil`**: 让你能直接运行 `mvn clean install` 或 `git commit`。
*   **`JSONUtil`**: 处理 Claude 那层层嵌套的 JSON 结构比原生 Jackson 或 Gson 稍微快一点点。

#### 3. 模拟 Claude Code 的“思维链”
如果你使用的是 **Claude 3.7 Sonnet**，你可以在请求中加入：
```java
body.set("thinking", new JSONObject().set("type", "enabled").set("budget_tokens", 2048));
```
这样 Claude 会先在内部思考如何重构你的代码，然后再决定调用哪些工具，效果非常接近官方的 CLI 工具。

### 建议实现的工具列表 (对应 Claude Code 的能力)
为了让你的 `ClaudeCodeUtil` 真的好用，建议至少实现以下四个工具：
1.  `ls`: 使用 `FileUtil.ls(path)` 查看目录。
2.  `read_file`: 使用 `FileUtil.readUtf8String(path)`。
3.  `write_file`: 使用 `FileUtil.writeUtf8String(content, path)`。
4.  `execute_command`: 使用 `RuntimeUtil.execForStr(cmd)`（慎用，建议做白名单限制）。
